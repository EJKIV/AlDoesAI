# Webby — User Guide

**Your webmaster.** Quiet, competent, keeps your public face polished.

## Daily Rhythm

**6:00 AM — Morning Maintenance**
- Syncs screenshots from ~/Documents/AlDoesAI/screenshots
- Rebuilds dashboard HTML with latest content
- Updates statistics (files, lines, prompts)
- Commits changes to Git
- Reports status

**Evening (if needed)**
- Checks if any urgent updates required
- Verifies all systems green

## What Webby Does Auto

✅ **Screenshot sync** — New images → assets/screenshots  
✅ **Dashboard rebuild** — Embeds all content fresh  
✅ **Git commit** — With descriptive timestamp  
✅ **Health check** — Verifies images load, links work  
✅ **Stats update** — File counts, line counts, timestamps  
✅ **Log maintenance** — Daily logs, weekly summaries  

## What Requires Your Approval

⏸️ **Major redesign** — Structural changes to layout/theme  
⏸️ **New features** — Anything beyond current scope  
⏸️ **New integrations** — Analytics, external services  
⏸️ **Domain changes** — Beyond content updates  

## How to Talk to Webby

**Direct:** "Webby, check the dashboard" → runs diagnostics  
**Task:** "Webby, add a new section for X" → proposes plan, awaits approval  
**Status:** "Webby, report" → gives morning summary anytime  
**Urgent:** "Webby, site looks broken" → drops everything, investigates

## Emergency Contacts

**If Webby reports issues:**
- Check `dashboard/index.html` directly
- Review `.agents/webby/logs/latest.log`
- Run `showmeal` to regenerate
- Message Al if Webby can't resolve

## Success Metrics

- [ ] Dashboard loads in <2 seconds
- [ ] All 10 screenshots display
- [ ] Conversation log renders inline
- [ ] Documents clickable/readable
- [ ] No broken image links
- [ ] Daily commits at 6 AM
- [ ] 99.9% uptime (the 0.1% is scheduled maintenance)
